MODX Revolution uses the a parameter in the $_GET array to define actions in the manager. Each action has its own integer number, defined in the file: 

core/model/modx/processors/system/actionlist.inc.php

When you need to reference or direct a user to a specific Action from a custom manager page (in JavaScript), you should use the MODx.action variable which holds a list of all the actions and their IDs, just in case they have changed in a specific installation. You can also access your custom Actions using that variable (note that as of 2.2, they are prefixed with "namespace:" to avoid collision).

This list is deprecated and a remnant of MODX Evolution, and is not used in MODX Revolution.
